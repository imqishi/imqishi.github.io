Rules of Machine Learning: Best Practices for ML Engineering — Martin Zinkevich

术语

下列术语将会在我们接下来的讨论中不断出现：

实例（Instance）：即想要预测的东西。例如，一个实例可以是一个你想要划分是“与猫相关”还是“与猫无关”的网页。

标记（Label）：一个预测任务的结果 — 可能是由机器学习系统产生的结果，或者是训练数据提供的正确答案。例如，一个网页的标记可以是“与猫相关”。

特征（Feature）：预测任务使用的实例中的一个属性。例如，一个网页可能有一个特征是“含有‘猫’这一单词”。

特征列（Feature Column）：相关特征的集合，如用户可能居住的所有可能的国家的集合。一个例子可能含有特征列中的一个或多个特征。特征列在VM系统（如在Yahoo或Microsoft）或字段（field）中可能被称作命名空间。

例子（Example）：一个实例（带有他的一系列特征）和一个标记。

模型（Model）：一个预测任务的统计表现。在例子上训练一个模型然后使用模型做出预测。

度量（Metric）：一个你关心的数字。可能被直接优化，也可能不行。

目标（Objective）：你的算法尝试优化的一个度量。

管道（Pipeline）：围绕一个机器学习算法的一些工具。包括从前端收集数据，把数据编入训练数据文件，训练一个至多个模型，并且将模型导出到生产中。

概览

你将面临的大多数问题都是工程问题。即便是一个有着巨大资源的伟大的机器学习专家，大多数的成果也是来自有用的特征，而不是厉害的机器学习算法。所以基本的方法是：

  1. 确保你的管道是端到端有效可用的
  2. 从一个有意义的目标开始
  3. 以简单的方法增加常识性特征
  4. 确保你的管道一直坚固有效

这个方法将会在相当长的时间耗费大量的金钱以及很多人的快乐。只有当没有更简单的技巧让你更远的时候，才能从这种方法开始。 增加复杂性会耗费更长的时间。

一旦你用尽了简单的技巧，未来可能确实存在更前沿的机器学习。 请参阅第三阶段机器学习项目。

本文档准备了四个部分：

  1. 第一部分帮助你理解什么情况下适用机器学习
  2. 第二部分是关于如何部署你的第一个管道
  3. 第三部分是关于启动和迭代，同时在管道中添加新功能，以及如何评估模型和训练策略
  4. 最后一部分关于达到瓶颈时你应该怎么做
  5. 之后是一些本文档用到的例子的相关背景、相关工作和附录
